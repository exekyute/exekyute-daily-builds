How does this work? click to expand click to collapse

1. Drop an invoice

Drag a PDF, PNG, or JPG into the box below. Files never leave your browser except for the optional Gemini call you opt into.

2. Tesseract reads it

A local OCR engine pulls text out of the page and reports how confident it is in what it read.

3. Gemini interprets it

If OCR confidence is high, Gemini reads the text. If confidence is low, Gemini sees the original image instead. Either path returns clean structured fields.

4. Sort, filter, export

Parsed invoices populate the table below. Sort by any column, filter live, watch the totals and charts update, then click Export CSV when you are done.

Privacy

Your Gemini API key is held only in memory for this tab. Refresh the page or click Reset Session and it is wiped. The key is never written to disk and is sent only to Google's Gemini API.

The table below contains sample invoices, not data from your documents. Each click of a mock button adds more rows on top of whatever is already there. Click Reset Session (or refresh the page) before processing real invoices so the mock totals do not mix with your actual data.

Mock: Clean adds 6 fully-parsed sample rows (random subtotals between $30 and $250, flat 14% tax). Use this to test sorting, filtering, charts, and CSV export.

Mock: Messy adds 4 deliberately broken sample rows that demonstrate how the table renders missing or invalid fields. Includes a wrong-length PO, an unreadable vendor, an illegible date, and a fully blank invoice. Use this to preview the N/A pills and the "Review manually" hint tooltip.

Reset Session wipes all invoices, filters, sort, and all in-memory settings (API key, PO digit length, confidence threshold). Equivalent to a browser refresh. Nothing is ever saved to disk, so there is nothing to "undo."

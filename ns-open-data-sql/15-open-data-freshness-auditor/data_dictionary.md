# Data dictionary

Two files. The audit report is the golden-verified output; the mart is the per-asset table the Power BI guide connects to.

## out/freshness_audit.csv (and expected/freshness_audit.csv)

A stacked report: 80 rows across five sections sharing one column set. Every value is text in the file; unused cells are empty. The `section` column says which reading applies.

| # | Column | Meaning |
| --- | --- | --- |
| 1 | `section` | Which block the row belongs to: `overall`, `bucket_summary`, `by_category`, `by_owner`, or `worst_offenders`. |
| 2 | `item` | The row's key: `all assets` (overall), the bucket name, the category, the department, or the asset name (worst_offenders). |
| 3 | `detail` | Only used by worst_offenders: the owning department of the named asset. Empty elsewhere. |
| 4 | `n_assets` | Asset count for the row's group. Empty for worst_offenders rows, which describe a single asset. |
| 5 | `n_stale_dormant` | How many of the group's assets are stale or dormant (`age_months >= 12`). Filled for overall, by_category, and by_owner. |
| 6 | `pct` | One-decimal percent. For overall, by_category, and by_owner: the share of the group's assets that are stale or dormant. For bucket_summary: the bucket's share of all 1,269 assets. |
| 7 | `age_months` | One-decimal average age of the group's dated assets in months. For worst_offenders: the single asset's own whole-number age. Empty when a bucket has no dated assets. |
| 8 | `last_updated` | Only used by worst_offenders: the asset's last data update date (`YYYY-MM-DD`). Empty elsewhere. |
| 9 | `row_rank` | Position of the row inside its section: bucket age order for bucket_summary, stale-share rank for by_category and by_owner, oldest-first rank for worst_offenders. |

Section sizes in this snapshot: 1 overall row, 5 bucket rows, 31 categories, 28 departments, 15 worst offenders.

## bi/exports/mart_freshness.csv

One row per catalogued asset, 1,269 rows, ordered oldest first. Generated by 99_export.sql into out/ and copied here by `run.py`.

| # | Column | Type | Meaning |
| --- | --- | --- | --- |
| 1 | `uid` | text | Socrata 4x4 id, extracted from the tail of the asset's portal URL. Unique per row. |
| 2 | `name` | text | Asset title as shown on the portal. |
| 3 | `type` | text | Asset kind, lowercased: `dataset`, `chart`, `map`, `filter`, `href`, `story`, or `calendar`. |
| 4 | `category` | text | Portal subject category; `(uncategorized)` when the source field is blank. |
| 5 | `owner` | text | Owning department or agency (the source's `detailedmetadata_department`); `(no department)` when blank. |
| 6 | `last_updated` | date | Last data update date, falling back to the last metadata update date when the data date is missing. Empty only when neither parses. |
| 7 | `age_months` | integer | Calendar month boundaries between `last_updated` and the pinned pull date 2026-07-06. Empty when `last_updated` is empty. |
| 8 | `bucket` | text | Freshness bucket: `Fresh` (under 6 months), `Aging` (6 to 11), `Stale` (12 to 23), `Dormant` (24 and up), `No date`. |
| 9 | `bucket_order` | integer | 1 through 5 in the bucket order above. Used as the sort-by column in Power BI so buckets never sort alphabetically. |
| 10 | `stale_or_dormant` | integer | 1 when `age_months >= 12` (Stale or Dormant), else 0. The flag behind the headline percent and the BI measures. |

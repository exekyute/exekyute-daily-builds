# Data dictionary

## out/mart_capital.csv

One row per capital-project record: one project at one location in one budget
year. 2,650 rows. This is also the frozen BI mart (`bi/exports/mart_capital.csv`)
and the source of `dashboard/data.js`.

| # | Column | Type | Meaning |
| --- | --- | --- | --- |
| 1 | `proj_no` | text | Project number. Repeats across locations and years (280 distinct). |
| 2 | `proj_name` | text | Project name, whitespace-normalized. |
| 3 | `loc_desc` | text | Location description (street, park, address). Blank on some records. |
| 4 | `work_desc` | text | Description of the capital work, whitespace-normalized. |
| 5 | `category` | text | Raw budget category exactly as published (whitespace-normalized). |
| 6 | `category_norm` | text | Normalized category folding the raw label duplicates. See spec.md. |
| 7 | `asset_type` | text | Asset type, whitespace-normalized. Blank becomes `(unspecified)`. |
| 8 | `year` | integer | Budget year (2013 to 2021). |
| 9 | `lat` | number | Latitude, WGS84, from the GeoJSON point, six decimals. |
| 10 | `lon` | number | Longitude, WGS84, from the GeoJSON point, six decimals. |

## out/counts_by_category_year.csv

Project counts on the category-by-year grid. 65 rows.

| # | Column | Type | Meaning |
| --- | --- | --- | --- |
| 1 | `category_norm` | text | Normalized category. |
| 2 | `year` | integer | Budget year. |
| 3 | `projects` | integer | Count of project records in that category and year. |

Row order: `category_norm`, then `year`.

## out/counts_by_asset_type.csv

Project counts by asset type. 15 rows.

| # | Column | Type | Meaning |
| --- | --- | --- | --- |
| 1 | `asset_type` | text | Asset type, or `(unspecified)` when blank. |
| 2 | `projects` | integer | Count of project records with that asset type. |

Row order: `projects` descending, then `asset_type`.

## out/category_ranking.csv

Normalized categories ranked by project count. 16 rows.

| # | Column | Type | Meaning |
| --- | --- | --- | --- |
| 1 | `category_rank` | integer | Dense rank by `projects` descending. 1 is the largest category. |
| 2 | `category_norm` | text | Normalized category. |
| 3 | `projects` | integer | Count of project records in that category. |
| 4 | `pct_of_total` | number | Share of all 2,650 records, one decimal. |

Row order: `category_rank`, then `category_norm`. Ties share a rank
(rank 13 twice at 2 projects, rank 14 twice at 1 project).

## dashboard/data.js

The mart re-emitted as a JavaScript array named `DATA`, one object per record,
carrying `proj_no`, `proj_name`, `loc_desc`, `category_norm`, `asset_type`,
`year`, `lat`, `lon`. Regenerated by `run.py`; do not edit by hand. The
dashboard re-derives every figure from this array, so the totals, the category
ranking, and the asset-type counts equal the golden CSVs above.
